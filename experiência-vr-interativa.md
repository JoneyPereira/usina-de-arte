# Descrição Geral

Nesta atividade, você vai dar um passo além da configuração básica e criar uma experiência em Realidade Virtual com identidade própria.
O projeto deve representar um ambiente temático inserido no contexto de Metaverso — como um museu virtual, uma simulação industrial, sala de aula, galeria de arte, área de treinamento, um hospital, uma sala de reuniões ou qualquer outro espaço que tenha um objetivo claro de uso.
Além de montar o ambiente, você deverá implementar ao menos uma interação funcional coerente com o tema escolhido, demonstrando que a experiência vai além de uma cena estática.

## Objetivo

Demonstrar compreensão dos fundamentos de desenvolvimento XR com Unity, aplicando-os na criação de um ambiente VR temático, interativo e documentado — conectando teoria e prática no contexto do Metaverso.

## Requisitos do Projeto

1. Configuração Técnica

- Projeto Unity criado com versão compatível com Meta XR SDK
- Meta XR SDK corretamente instalado e configurado
- Build Settings preparados para plataforma Android (Meta Quest)
- XR Plugin Management configurado adequadamente
- A movimentação inicial deve funcionar no PC (Editor), sem depender exclusivamente do óculos de VR

2. Ambiente Virtual

- Mínimo de 5 objetos 3D na cena (primitivos ou assets importados)
- Um plano de chão ou terreno navegável-Skybox configurado (céu ou ambiente ao redor)
- Objetos posicionados de forma coerente, criando um ambiente reconhecível
- O ambiente deve representar um contexto específico de Metaverso, a suaescolha, como discutido durante as aulas e mentorias
- O projeto deverá ter pelo menos 1 (uma) interação com algum dos objetos,explicado melhor no próximo ponto

3. Interação Obrigatória
O projeto deve conter ao menos uma interação funcional implementada em C#,coerente com o contexto do ambiente. Exemplos aceitos:

- Objeto interagível que emite som ao ser tocado ou clicado
- Elemento que muda de cor, tamanho ou material ao ser ativado
- Objeto que inicia ou para uma animação ao ser interagido
- Painel que exibe informação ao se aproximar-Porta ou gaveta que abre/fecha por interação
- Outra interação aprovada pelo mentor, desde que coerente com o tema
Dica: a interação deve ter relação com o objetivo do ambiente. Ex: em um museu, tocar em um quadro para ouvir uma narração; em um hospital, interagir com um equipamento para ver uma animação de funcionamento.

4. Organização e Qualidade

- Hierarquia de objetos organizada com grupos lógicos
- Nomenclatura clara e consistente de todos os elementos
- Projeto limpo, sem arquivos desnecessários
- Scripts C# comentados, explicando a lógica das interações

5. Documentação — README no Repositório
O repositório no GitHub deve conter um arquivo README.md (ouREADME.txt) com os seguintes tópicos obrigatórios:

- Nome Completo do(a) aluno(a) (Se preferir, o aluno pode adicionar seunome direto no arquivo junto do link que será enviado no Homero,divulgando apenas na plataforma)
- Apresentando o Seu Projeto — descrição do que foi criado
- Contexto e Objetivos — qual problema ou objetivo o ambiente representano Metaverso (educação, entretenimento, indústria, comunicação)
- Processo de Criação e Dificuldades — como você desenvolveu o projeto,quais foram os maiores desafios e como os resolveu

6. Entregas Necessárias

- Projeto Unity completo em repositório público no GitHub
- Incluir apenas as pastas necessárias: Assets, ProjectSettings, Packages,Scenes (Não faça upload dos arquivos não necessários, utilize umarquivo .gitignore para Unity)
- O projeto deve ser enviado à plataforma Homero como um arquivo .txt contendo o link do repositório no GitHub
- O nome completo do aluno deve estar no arquivo de envio ou no README do repositório
